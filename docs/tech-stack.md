# 技術スタック解説（初心者・非エンジニア向け）

今回のカレンダーアプリで使う技術を、できるだけ平易な言葉で説明します。
プログラミングを始めたばかりの方でも理解できるよう、例えを交えながら解説します。

---

## 全体のイメージ

Webアプリは大きく **「フロントエンド」** と **「バックエンド」** に分かれます。

```
【あなたのスマホ/PC】          【インターネット上のサーバー】
  ┌──────────────┐               ┌──────────────────────────┐
  │              │               │  Vercel（ホスティング）   │
  │  ブラウザ    │  ←→ 通信 →→  │  └ Next.js（フロント）    │
  │  (Chrome等)  │               │                          │
  │              │               │  Supabase（バックエンド） │
  └──────────────┘               │  └ データベース           │
                                  │  └ ログイン機能          │
                                  └──────────────────────────┘
```

- **フロントエンド** = 画面（あなたの目に見える部分）を作る技術
- **バックエンド** = データの保存・ログイン管理など、裏側の仕組み

## 1. HTML / CSS / JavaScript — Webの基本3つ

すべてのWebサイトはこの3つで成り立っています。

### HTML（HyperText Markup Language）
> **「骨格」** を作るもの

文章の構造を定義します。「ここが見出し」「ここがボタン」「ここが表」といった情報を書きます。

```html
<!-- 例：ボタンを作る -->
<button>予定を追加</button>
```

### CSS（Cascading Style Sheets）
> **「見た目」** を整えるもの

HTMLに色・サイズ・レイアウトなどのデザインを適用します。

```css
/* 例：ボタンを青くする */
button {
  background-color: blue;
  color: white;
}
```

カレンダーの「土日を幅広く・平日を幅狭く」もCSSで実現します。

```css
/* 日・月・火・水・木・金・土 の順に幅を設定 */
.calendar-grid {
  display: grid;
  grid-template-columns: 2fr 1fr 1fr 1fr 1fr 1fr 2fr;
}
```

`fr` は「比率」を表す単位です。`2fr` は `1fr` の2倍の幅になります。

### JavaScript
> **「動き」** を加えるもの

ボタンを押したときに何かが起きる、データを取ってきて表示する、といった「動的な処理」を書きます。

```javascript
// 例：ボタンを押したらメッセージを表示
button.addEventListener('click', () => {
  alert('予定を追加しました！');
});
```

## 2. React — 画面を「部品」で組み立てるライブラリ

> **「レゴブロック」** のように画面を作れるツール

HTMLを直接書くと、予定の数が増えたときに一つ一つ手で書き直す必要があります。
Reactを使うと「予定カード」という部品を一度だけ作れば、データを渡すだけで何個でも自動で表示してくれます。

```jsx
// 例：予定カードという部品（コンポーネント）を作る
function EventCard({ title, color }) {
  return (
    <div style={{ backgroundColor: color }}>
      {title}
    </div>
  );
}

// 使う側：データを渡すだけで表示される
<EventCard title="山田さんと打ち合わせ" color="#3B82F6" />
<EventCard title="家族旅行" color="#22C55E" />
```

**メリット**: コードの再利用・管理が楽になる、複雑なUIも整理しやすい

---

## 3. Next.js — Reactをさらに使いやすくしたフレームワーク

> **「完成済みの家の骨組み」** のようなもの

ReactだけではWebアプリとして動かすための「おまじない」をたくさん自分で書く必要があります。
Next.jsはそういった面倒な設定を最初から用意してくれています。

### Next.jsが自動でやってくれること
- URLごとに表示するページを切り替える（ルーティング）
- 表示を高速化する（サーバーサイドレンダリング）
- Vercel（後述）への公開が1コマンドで可能

### フレームワークとライブラリの違い

| | ライブラリ（React） | フレームワーク（Next.js） |
|--|--|--|
| イメージ | 便利な道具 | 道具セット＋使い方のルール |
| 自由度 | 高い | やや低いが迷わない |
| 初心者向き | 慣れが必要 | とっつきやすい |

## 4. TypeScript — 「型」を使ってミスを減らすJavaScript

> **「入力チェック付きのJavaScript」**

JavaScriptは自由度が高い反面、間違えてもすぐには気づけないことがあります。
TypeScriptは「この変数には数字しか入らない」「この関数は文字列を返す」とあらかじめ決めることで、書いている最中にミスを教えてくれます。

```typescript
// TypeScriptの例：引数の型を指定
function greet(name: string): string {
  return `こんにちは、${name}さん`;
}

greet(123); // ← エラー！ 数字は渡せない、と教えてくれる
greet("田中"); // ← OK
```

**メリット**: バグが減る、エディタの補完が強力になる

## 5. Tailwind CSS — クラス名でスタイルを当てるCSS

> **「スタイルの名前を直接HTMLに書く」** 方法

通常のCSSは別ファイルに書きますが、Tailwind CSSはHTMLに直接クラス名を書くことでスタイルを適用します。

```html
<!-- 通常のCSS（別ファイルが必要）-->
<button class="my-button">追加</button>
<!-- my-button { background: blue; padding: 8px 16px; ... } を別に書く -->

<!-- Tailwind CSS（HTMLに直接書く） -->
<button class="bg-blue-500 text-white px-4 py-2 rounded">追加</button>
```

**メリット**: CSSファイルを行き来しなくていい、よく使うスタイルを暗記しやすい

## 6. Supabase — データベース＋ログインをまとめたサービス

> **「データの倉庫＋鍵の管理」** を外部サービスに任せる

アプリのデータ（予定の一覧など）はどこかに保存しなければなりません。
また、「管理者だけ編集できる」という制限も必要です。
Supabaseはこれらをすべてまとめて提供してくれます。

### Supabaseが提供するもの

| 機能 | 説明 |
|------|------|
| **データベース** | PostgreSQLというデータベースをそのまま使える |
| **認証（Auth）** | メール＋パスワードのログイン機能を数行で実装可 |
| **RLS**（行レベルセキュリティ） | 「このデータはこの人だけ編集可」というルールを設定 |
| **APIの自動生成** | データベースのテーブルを作るだけでAPIが使えるようになる |

### PostgreSQL（データベース）とは

データをExcelの表のように管理するシステムです。

```
events（予定）テーブルのイメージ:

| id  | user_id | title          | date       | color   |
|-----|---------|----------------|------------|---------|
| 001 | abc     | 山田さんと面談 | 2026-03-21 | #3B82F6 |
| 002 | abc     | 家族旅行       | 2026-03-22 | #22C55E |
```

### Firebase との違い

Firebase（Google）も同様のサービスですが、SupabaseはSQLが使えて自由度が高く、無料枠も広いため今回採用します。

## 7. Vercel — Webアプリを無料で公開できるサービス

> **「作ったアプリをインターネットに置く棚」**

GitHubにコードを push するだけで、自動的にWebサイトとして公開してくれます。

### Vercelの特徴

| 特徴 | 説明 |
|------|------|
| 無料枠 | 個人利用なら基本無料 |
| 自動デプロイ | GitHubに push → 即座に公開 |
| HTTPS | 自動でセキュアな接続を設定 |
| 独自ドメイン | 無料サブドメイン（.vercel.app）が使える |

---

## 8. Git / GitHub — コードの変更履歴を管理するツール

### Git
> **「コードの変更履歴を記録するタイムマシン」**

コードを編集するたびに「セーブポイント」を作れます。失敗しても過去の状態に戻せます。

```
コミット履歴のイメージ:

○ 「ログイン機能を追加」 ← 最新
│
○ 「カレンダー表示を実装」
│
○ 「最初のコード」
```

### GitHub
> **「Gitで管理したコードをインターネット上に置く場所」**

チームで開発するときにコードを共有したり、Vercelがコードを取得するときに使います。

```
                            ↕ 通信
┌──────────────────────────────────────────────────────────┐
│  インターネット上のサーバー                               │

コード管理: Git / GitHub（変更履歴・バックアップ）
```
